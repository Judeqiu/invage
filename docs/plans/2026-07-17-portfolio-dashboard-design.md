# Portfolio Dashboard Report ‚Äî Design

**Status:** approved 2026-07-17  
**Repo:** invage (Invester domain agent)

## Problem

Users can save dated portfolio snapshots (`save_snapshot`) and a 3-axis analysis HTML report (`save_report`), but there is no single **dashboard report** that shows:

1. **Current value and P/L vs cost basis** (live)
2. **Value change over time** (from snapshot history)

They want this as a **generated HTML report** (BinDrive + signed URL), not a live WebUI page.

## Goals

- Generate a self-contained HTML **Portfolio Dashboard** report
- Hero metrics: position count, live total value, total P/L $/% vs cost
- Period change from the last two snapshots when history exists
- Value history table + inline SVG sparkline (no external CDN)
- Holdings table with weight % and P/L
- Fail-fast: no invented zeros, no silent skip of corrupt snapshots

## Non-goals (report v1)

- Auto-save snapshot when generating dashboard
- Allocation pie charts, 7d/30d filters

## WebUI tab (shipped)

Dynamic page + plugin via `DomainExtension.webUi` (no Utarus SPA fork):

- Nav **Dashboard** ‚Üí iframe ‚Üí `webui/dashboard/`
- API `GET /api/domain/invage/dashboard`
- See `docs/webui-chat-design.md`

## Product surface

| Section | Content |

| Header | User name, generated-at |
| Hero | Positions ¬∑ Live total value ¬∑ Total P/L $/% vs cost ¬∑ Period Œî when ‚â•2 snapshots (else empty-state text) |
| Value history | Snapshot date, total value, Œî$ / Œî% vs prior snapshot; SVG sparkline if ‚â•2 points |
| Holdings | Ticker, units, cost, price, value, weight %, P/L $/% sorted by value desc |
| Footer | Generated by Invester ¬∑ Yahoo Finance ¬∑ Not financial advice |

### Period change definition

- Snapshots ordered by `date` ascending
- Period change on hero = last snapshot `totalValue` ‚àí previous snapshot `totalValue` (and %)
- Live total may differ from last snapshot; label **Live value** vs **Last snapshot** clearly

### Data sources

| Block | Source |
|-------|--------|
| Live prices / P/L | User portfolio YAML + `fetchPrices` |
| History | `data/drive/<slug>/snapshots.json` + `snapshot-YYYY-MM-DD.json` |
| Display name | `profile.display_name` |

Dashboard does **not** auto-create a snapshot.

### Modules

| Piece | Path |
|-------|------|
| Snapshot types + loaders | `src/state/snapshot.ts` |

| HTML template | `src/report/dashboard-template.ts` |
| Tool | `save_report` with `kind?: "analysis" \| "dashboard"` (default `"analysis"`) |
| Snapshot tools | Refactor `src/tools/snapshot.ts` to use shared types |

### Tool behavior

- `kind: "analysis"` (default) ‚Äî existing 3-axis report; default file `report-YYYY-MM-DD.html`
- `kind: "dashboard"` ‚Äî dashboard report; default file `dashboard-YYYY-MM-DD.html`
- Optional custom `name` still overrides filename
- Tool result should note that regular `save_snapshot` improves period-change history

## Error handling

| Condition | Behavior |
|-----------|----------|
| Empty portfolio | Fail |
| Missing market price for a ticker | Fail naming ticker |
| Snapshot index lists a missing/corrupt file | Fail with path + reason |
| Invalid `kind` | Schema enum reject |
| No snapshots | Still succeed; history empty state, no period change |
