Core capability

3-axis analysis against live market data

Every holding is evaluated through three lenses. The agent fetches real-time prices and analyst targets from Yahoo Finance, then classifies each position with a specific recommendation. Not a snapshot — the analysis reflects the market at the moment you ask.

🔴
Laggards

Cost price is above even the highest analyst target. Underwater with limited recovery path. Hardest position to hold.

  • SellLoss > 30% — no recovery path
  • HoldLoss 15–30% — wait for catalyst
  • ReduceLoss < 15% — average down if fundamentals strong
🟡
Overpriced

Trading above the median analyst target. The market is pricing in more than the consensus expects. Time to think about protection.

  • Take profitPremium > 20% above median
  • Trail stopPremium 10–20% — protect gains
  • HoldPremium < 10% — momentum may continue
🟢
Buy opportunities

Trading at least 15% below the median analyst target. The consensus sees material upside from here. The cleanest signal.

  • Strong buyUpside > 30% — high conviction
  • BuyUpside 20–30% — moderate conviction
  • WatchUpside 15–20% — interesting, monitor
What the agent can do

Every action is a typed, validated tool call

The LLM can't invent data or skip steps — every portfolio mutation, market fetch, and report action is a tool call with a TypeBox schema that is validated before execution. If a tool refuses, the agent surfaces the error. No silent corruption.

add_holding Add or update a position with ticker, average cost, share count, and fund category.
remove_holding Remove a single position by ticker. Refuses if the ticker isn't in the portfolio.
update_holding Patch individual fields — change just the price, the share count, or the category.
get_portfolio Show every holding with cost basis, per-ticker P/L, and total portfolio cost.
clear_portfolio Wipe all positions. Requires the LLM to confirm with the user first (boolean guard).
portfolio_analyzer Run the full 3-axis analysis. Works on saved positions or ad-hoc tickers for a one-shot read.
save_report Generate an HTML portfolio report and drop it into the user's InvesterDrive folder.
send_report Auto-generate an analysis and email it to any address via Gmail.
save_snapshot Capture today's portfolio state with live prices. JSON file, append-only history.
list_snapshots Review every snapshot chronologically — see how P/L evolved over time.
bindrive_* Upload, download, list, and delete files in the user's personal drive through the agent.
use_skill Load domain knowledge (investment analysis framework, admin guide) on demand from markdown docs.
init_user Create a new user state file with display name, email, and auto-generated auth token.
link_telegram / unlink_telegram Bind or unbind a Telegram account so the agent auto-identifies who is talking.
issue_invite_code / redeem_invite_code Admin issues single-use invite codes. Redeeming creates the user accounts the framework.
invite / admincode / revoke Admin privilege management — onboard new admins without touching the server.
How it works

You talk. It reasons. It acts. It remembers.

1

You message the agent in plain language

"Add 50 shares of AAPL at $210 average cost"
"How is my portfolio doing?"
"Analyze my positions and email me the report"

2

The agent loads your state and your skills

YAML per-user state is the source of truth — no re-entering positions. Domain knowledge (the 3-axis framework, metrics guide) loads on demand from markdown skills.

3

It calls tools against live data

Real-time prices and analyst targets from Yahoo Finance. Every tool parameter is schema-validated. Refusal is surfacing, not silent.

4

State is mutated, logged, and persisted

Every change appends to an audit log with a timestamp. You can read any user file in a text editor. No database, no migrations, no black box.

5

You get a structured answer — and can push it further

HTML reports land in your InvesterDrive folder. Email delivery via Gmail. Snapshots capture performance history. All from the same conversation.

Architecture

Five layers · one conversational loop

Built on the Utarus framework + pi-agent-core. Each Telegram user gets an isolated agent with a 24-hour context window. Tools are the only way the LLM touches state — it can't bypass them.

What you would actually use it for

Four workflows from one agent

Build a portfolio over time Add positions trade by trade. The agent tracks your cost basis across Update calls. Snapshots capture performance on every check-in.
Run a periodic review "Analyze my portfolio" pulls live prices, runs 3-axis analysis, and tells you exactly what's broken. No spreadsheet needed.
Share a report with a partner One command generates an HTML report and emails it via Gmail. The same report is viewable in the InvesterDrive web portal.
Let a client self-serve Invite codes + per-user isolation. Each client gets their own agent context, their own state, their own reports. They never see each other.
Run it

Self-hosted in three commands

Node 20+ npm DeepSeek API key Telegram bot token (optional)
$ git clone https://github.com/Judeqiu/invage.git && cd inserter
$ npm install
$ cp .env.example .env # fill DEEPSEEK_API_KEY, UTARUS_AGENT_NAME, UTARUS_AGENT_PURPOSE
$ npm run dev
